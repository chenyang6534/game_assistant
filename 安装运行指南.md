# WindowPilot 安装运行指南

这份文档面向两类使用方式：

- 源码运行：适合开发、调试和频繁修改配置
- 打包运行：适合分发给没有 Python 环境的电脑

## 先看结论

如果你只想尽快跑起来，按这个顺序做：

1. 安装 Python 3.9+
2. 打开项目目录
3. 双击“安装依赖.bat”
4. 双击“快速启动.bat”

如果你拿到的是已经打包好的输出目录，则直接运行 dist\WindowPilot\WindowPilot.exe 即可。

## 运行前提

### 系统要求

- Windows 10/11，建议 64 位系统
- Python 3.9 或更高版本
- 建议至少 4GB 内存
- 建议预留 1GB 以上可用磁盘空间

### 功能相关前提

- 后台输入、部分窗口附加、内存扫描，通常需要管理员权限
- 网络消息页如果需要完整双向抓取，建议安装 Npcap
- 可选 AI 目标识别工作区需要额外安装 ai_tile_mvp/requirements-ai.txt

## 方式一：源码运行

### 步骤 1：安装 Python

1. 访问 https://www.python.org/downloads/
2. 安装 Python 3.9 或更高版本
3. 安装时勾选“Add Python to PATH”

### 步骤 2：确认 Python 可用

打开 PowerShell 或命令提示符，执行：

```bash
python --version
```

如果能看到版本号，就可以继续。

### 步骤 3：进入项目目录

示例：

```bash
cd D:\pytest1\game_assistant
```

### 步骤 4：安装依赖

你可以任选一种方式：

方式 A：使用脚本

```bat
安装依赖.bat
```

方式 B：手动安装

```bash
pip install -r requirements.txt
```

如果你更习惯隔离环境，建议先创建虚拟环境：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 步骤 5：启动程序

方式 A：使用脚本

```bat
快速启动.bat
```

方式 B：命令行运行

```bash
python main.py
```

如果涉及后台输入、内存扫描或某些窗口控制，建议以管理员权限启动。

## 方式二：使用打包后的 EXE

### 适用场景

- 目标电脑没有 Python 环境
- 需要交付给其他人直接使用
- 不希望对方手动安装依赖

### 使用方法

1. 拿到 dist\WindowPilot 整个目录
2. 将整个目录复制到目标电脑，例如 D:\Tools\WindowPilot
3. 双击 WindowPilot.exe 启动
4. 若某些功能无效，再尝试右键以管理员身份运行

### 分发时的注意点

打包脚本默认会带上程序运行所需代码和基础配置，但这些内容如果你要一并交付，需要手工确认：

- 你自己的 tasks 任务文件
- 你自己的 pic 图片资源
- coordinate_mappings.json、coordinate_workspace_state.json 等坐标映射数据
- coordinate_workspace_assets 目录
- 可选 AI 模型文件和相关元数据

如果只是分发一个“空白可运行版本”，可以不复制这些用户数据。

## 启动脚本说明

项目根目录里常用脚本如下：

- 安装依赖.bat：检查 Python、升级 pip、安装 requirements.txt
- 快速启动.bat：检查核心依赖后启动 main.py
- 打包成EXE.bat：完整打包流程
- 简易打包.bat：更轻量的打包流程
- 清理打包文件.bat：清理 build、dist、__pycache__

## 常见问题

### 1. 提示“Python 不是内部或外部命令”

原因通常是 Python 未安装，或安装时没有加入 PATH。

处理方法：

1. 重新安装 Python
2. 安装时勾选“Add Python to PATH”
3. 安装后重新打开终端再试

### 2. 安装依赖失败

先执行：

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

如果网络较慢，可改用镜像源：

```bash
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
```

### 3. 找不到目标窗口

建议按这个顺序排查：

1. 确认目标程序已经打开
2. 在主界面点击“刷新”
3. 尽量先使用窗口化而不是独占全屏
4. 若目标程序权限更高，尝试以管理员身份启动本工具

### 4. 图像识别不稳定

建议优先检查：

1. 模板图片是否清晰
2. 阈值是否过高
3. 是否应该改用“前景优先匹配”
4. 是否需要勾选颜色一致性校验
5. 截图区域和缩放比例是否发生变化

### 5. 文字识别结果为空

可以依次检查：

1. OCR 依赖是否已正常安装
2. 当前截图中是否确实有清晰文字
3. 最小置信度是否设得过高
4. 后台截图是否对当前目标无效，必要时切换前台验证

### 6. 网络消息页只能看到发送方向

这通常不是程序故障，而是没有安装 Npcap。

处理方法：

1. 安装 Npcap
2. 重启程序
3. 重新开始抓取

### 7. 内存扫描无法附加

优先检查：

1. 当前是否以管理员身份运行
2. 目标 PID 是否正确
3. 目标进程是否受额外保护

## 更新程序

### 源码方式

直接用新版本文件覆盖旧版本，然后重新执行依赖安装即可。

### EXE 方式

重新打包后，整体替换 dist\WindowPilot 目录。

如果你要保留历史任务、图片或坐标文件，请先备份对应目录和 JSON 文件。

## 建议保留的排查入口

遇到问题时，优先看这几处：

1. logs 目录中的日志
2. config.json 中的热键和窗口配置
3. coordinate_mappings.json 与 coordinate_workspace_state.json
4. tasks 目录中的任务文件

## 使用边界

请仅在你有权控制、调试或测试的目标程序和环境中使用本工具。不同窗口的权限级别、渲染方式、输入处理和安全策略可能不同，实际效果需要逐项验证。
