# 打包错误排查

这份文档按“现象 -> 原因 -> 处理顺序”整理，优先覆盖当前仓库最常见的打包问题。

## 先做这 4 步

如果你现在没有时间分析，先直接按这个顺序操作：

1. 关闭所有 WindowPilot.exe
2. 用普通用户打开终端，不要用管理员终端
3. 运行“清理打包文件.bat”
4. 重新执行“打包成EXE.bat”

很多问题做到这里就解决了。

## 现象 1：Permission denied / 文件被占用

### 常见报错

```text
PermissionError: [Errno 13] Permission denied: '...\dist\WindowPilot\_internal\ucrtbase.dll'
```

### 最常见原因

- 正在运行旧的 WindowPilot.exe
- 资源管理器或杀毒软件占用了 dist 或 build 下的文件
- 上一次打包未完全退出

### 处理顺序

1. 打开任务管理器，结束所有 WindowPilot.exe
2. 执行“清理打包文件.bat”
3. 若仍失败，手动删除 build 和 dist
4. 若手动也删不掉，重启电脑后第一时间重新打包

## 现象 2：提示不要以管理员身份运行 PyInstaller

### 常见报错

```text
DEPRECATION: Running PyInstaller as admin is not necessary nor sensible.
```

### 原因

PyInstaller 新版本不建议在管理员权限下执行打包。

### 正确做法

- 打包时：普通用户运行
- 运行打包结果时：按需决定是否管理员启动

## 现象 3：从错误目录运行

### 常见报错

```text
Error: Do not run pyinstaller from C:\Windows\system32
```

### 原因

当前工作目录不在项目根目录，脚本或命令找不到正确资源。

### 处理方法

先进入项目目录，再执行任何打包命令：

```powershell
cd D:\pytest1\game_assistant
```

## 现象 4：提示找不到 PyInstaller

### 处理方法

```bash
pip install pyinstaller
```

如果你在虚拟环境中工作，先激活虚拟环境再装。

## 现象 5：提示缺少模块或运行库

### 打包阶段缺少模块

先重新安装依赖：

```bash
pip install -r requirements.txt
```

再重新打包。

### 打包后启动时缺少模块

通常有两种原因：

1. 你用了过于简化的 PyInstaller 命令，没带 hidden-import 或 collect-all
2. 你手动打包时没有带上 templates、config.json 等资源

建议直接改用“打包成EXE.bat”，不要自己再缩减参数。

## 现象 6：打包成功，但启动后直接退出或没有界面

### 先检查

1. 是否能用源码方式正常运行 python main.py
2. dist\WindowPilot\_internal 是否存在
3. 打包时是否使用了当前仓库的完整参数

### 更稳妥的排查方式

可以暂时不用无控制台模式，改成手动加上 console 调试，先看异常输出。

如果源码方式也启动不了，就不是打包问题，应先修源码运行环境。

## 现象 7：程序能启动，但自己的任务、图片或坐标数据不见了

### 原因

当前打包脚本默认只保证程序主体可运行，不会自动把你的所有用户数据都打进去。

### 需要手工复制的常见内容

- tasks
- pic
- coordinate_mappings.json
- coordinate_workspace_state.json
- coordinate_workspace_assets
- 你自己的日志、模板或预设资源

如果要分发“带配置”的版本，打包完成后请把这些内容手动复制到 dist\WindowPilot 中。

## 现象 8：AI 识别相关功能在打包后不可用

### 常见原因

- 没有复制 onnx 模型和对应 json 元数据
- 没有复制 ai_tile_mvp 相关模型目录
- 任务里引用的是开发环境中的绝对路径

### 处理方法

1. 确认模型文件实际存在
2. 确认打包输出目录中也有这些模型
3. 优先改成相对路径或重新在目标环境中选择模型路径

## 现象 9：网络消息页只能抓到发送方向

这通常不是打包失败，而是目标电脑没装 Npcap。

处理方法：

1. 安装 Npcap
2. 重启程序
3. 再测试抓包

## 现象 10：清理脚本也删不掉文件

### 可能原因

- 文件仍被进程占用
- 杀毒软件正在扫描
- 资源管理器停留在 dist 目录中

### 处理顺序

1. 关闭资源管理器中所有打开的 dist 和 build 窗口
2. 关闭正在运行的程序
3. 暂时关闭实时防护或加入白名单
4. 再次执行清理脚本
5. 仍不行就重启系统后再删

## 杀毒软件干扰怎么处理

如果每次都在复制 DLL 或生成 exe 时失败，优先怀疑安全软件拦截。

可选做法：

1. 临时关闭实时防护
2. 将项目目录加入白名单
3. 打包完成后再恢复防护

## 一份稳妥的手动重试流程

```powershell
cd D:\pytest1\game_assistant
taskkill /F /IM WindowPilot.exe 2>$null
if (Test-Path build) { Remove-Item -Recurse -Force build }
if (Test-Path dist) { Remove-Item -Recurse -Force dist }
python -m PyInstaller --name=WindowPilot --windowed --onedir --noconfirm --add-data="templates;templates" --add-data="config.json;." --hidden-import=win32gui --hidden-import=win32api --hidden-import=win32con --hidden-import=pywintypes --hidden-import=win32clipboard --hidden-import=pynput.keyboard._win32 --hidden-import=pynput.mouse._win32 --collect-all=PySide6 --collect-all=rapidocr_onnxruntime main.py
```

## 最后兜底

如果前面的办法都不行：

1. 重启电脑
2. 不启动任何已打包程序
3. 用普通用户打开终端
4. 先确认源码模式能正常运行
5. 再重新执行打包

## 需要反馈时请附带的信息

至少提供这些内容：

```powershell
python --version
pip show pyinstaller
Get-Location
```

再附完整报错文本，以及你执行的是哪个脚本或命令。
