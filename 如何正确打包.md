# 如何正确打包

这份文档只讲当前仓库推荐的打包方式，不再沿用旧版的简化命令说明。

## 先看结论

当前最稳妥的做法是：

1. 用普通用户打开终端，不要用管理员身份
2. 进入项目根目录
3. 确认 WindowPilot.exe 没有在运行
4. 优先执行“打包成EXE.bat”
5. 打包完成后分发整个 dist\WindowPilot 目录

## 打包前必须知道的规则

### 1. 不要以管理员身份运行打包

PyInstaller 7.x 对管理员运行更敏感，很多环境下会直接给出警告或失败。

正确做法：

- 打包命令用普通用户运行
- 打包后的程序，如果某些功能需要更高权限，再按需以管理员身份启动

### 2. 一定要在项目目录下执行

推荐目录应类似：

```text
D:\pytest1\game_assistant
```

不要在 system32 或其他临时目录里直接运行打包脚本。

### 3. 先关闭正在运行的程序

如果 dist\WindowPilot\WindowPilot.exe 还在运行，旧文件会被锁住，导致清理或覆盖失败。

### 4. 尽量在稳定的 Python 环境里打包

建议做法：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
pip install pyinstaller
```

## 推荐方式一：使用完整打包脚本

这是当前最推荐的方式，脚本已经包含：

- 清理旧 build 和 dist
- 结束正在运行的 WindowPilot.exe
- 自动检查并安装 PyInstaller
- 补充当前项目需要的 hidden imports 和 collect-all

### 步骤

1. 打开 PowerShell 或命令提示符
2. 切换到项目目录
3. 运行：

```bat
打包成EXE.bat
```

### 输出位置

成功后输出目录为：

```text
dist\WindowPilot\WindowPilot.exe
```

## 推荐方式二：使用简易打包脚本

如果你只想快速验证打包是否能跑通，可以执行：

```bat
简易打包.bat
```

它比完整脚本更轻量，但本质仍然是 onedir 打包。

## 手动打包方式

如果你不想通过批处理脚本，也可以手动执行与当前脚本接近的命令。

先进入项目目录，再运行：

```powershell
python -m PyInstaller `
    --name=WindowPilot `
    --windowed `
    --onedir `
    --add-data="templates;templates" `
    --add-data="config.json;." `
    --hidden-import=win32gui `
    --hidden-import=win32api `
    --hidden-import=win32con `
    --hidden-import=pywintypes `
    --hidden-import=win32clipboard `
    --hidden-import=pynput.keyboard._win32 `
    --hidden-import=pynput.mouse._win32 `
    --collect-all=PySide6 `
    --collect-all=rapidocr_onnxruntime `
    --noconfirm `
    main.py
```

如果你在 cmd 中执行，请改成批处理换行符 `^`。

## 打包输出里默认包含什么

当前脚本会带上：

- 主程序代码
- PySide6 相关运行库
- rapidocr-onnxruntime 相关运行文件
- templates 目录
- config.json

## 哪些内容需要你自己复制

如果你想分发“带你自己的配置和资源”的版本，通常还需要手工复制这些内容到 dist\WindowPilot：

- tasks 目录中的任务文件
- pic 目录中的图片资源
- coordinate_mappings.json
- coordinate_workspace_state.json
- coordinate_workspace_assets 目录
- logs 中你想保留的日志
- 可选 AI 模型、元数据和相关资源

如果只是分发基础运行版本，这些内容可以不带。

## 打包完成后的检查项

至少确认以下几点：

1. dist\WindowPilot\WindowPilot.exe 存在
2. dist\WindowPilot\_internal 目录存在
3. 双击 WindowPilot.exe 能启动主界面
4. 可以正常选择窗口
5. 图像识别或 OCR 至少有一项能正常工作

## 分发建议

### 正确的分发方式

复制整个目录：

```text
dist\WindowPilot
```

不要只拿出单个 exe 文件。

### 目标电脑上的运行建议

- 先直接双击运行
- 若后台输入、内存扫描等功能不正常，再尝试管理员身份运行
- 如需完整网络抓取，单独安装 Npcap

## 打包时最常见的错误操作

不要这样做：

- 右键“以管理员身份运行”打包脚本
- 在 system32 目录里启动打包
- 一边运行程序一边重新打包
- 没装依赖就直接执行 PyInstaller
- 只拷贝 exe，不拷贝整个输出目录

## 一份最小检查清单

打包前：

- 当前目录正确
- Python 可用
- PyInstaller 已安装
- 没有管理员终端
- 没有正在运行的 WindowPilot.exe
- dist 和 build 没有被占用

打包后：

- exe 存在
- 主界面能打开
- 关键功能至少验证一项

## 如果你只是要快速重试

按这个顺序做：

1. 关闭所有 WindowPilot.exe
2. 运行“清理打包文件.bat”
3. 用普通用户重新执行“打包成EXE.bat”

## 需要提供排查信息时

如果打包仍然失败，请保留以下信息：

```powershell
python --version
pip show pyinstaller
Get-Location
```

再附上完整报错文本，而不是只发最后一行。
