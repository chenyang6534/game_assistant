# 地块标注实操清单

这份清单只做一件事：让你从截图到可训练数据，按最短路径走通。

下面所有路径都默认以当前这个目录为基准：

- ai_tile_mvp/

## 1. 这次改成什么方案

不再做 64 类硬标。

现在改成两步：

1. 只标 1 个检测类：resource_node
2. 每个框再挂 3 个属性：等级、类型、关系

这样做好处很直接：

- 你画框时不用在 64 类里翻来翻去
- 检测模型只学“哪里有菱形”
- 等级、类型、关系改成三个小分类任务
- 标注和修框成本都会低很多

## 2. 本项目要标什么

检测标签只保留 1 类：

- resource_node

框里要包含完整菱形主体，不要只框数字。

每个框再补 3 个属性：

- 等级：4级、5级、6级、7级、8级、9级、10级
- 类型：木材、石头、铁矿、铜矿、粮食
- 关系：同盟、友盟、中立、敌对、我方

## 3. 本项目固定目录

原图目录：

- datasets/plot_det/raw/images

AnyLabeling JSON 标签目录：

- datasets/plot_det/raw/labels

检测类别文件：

- configs/label_classes.txt

属性配置文件：

- configs/resource_node_attributes.json

属性分类数据集根目录：

- datasets/plot_attr_cls

抽检输出目录：

## 4. 采样步骤

如果想用交互方式采样：

```powershell
python ai_tile_mvp/scripts/sample_map_tiles_interactive.py
```

如果想直接用批处理启动：

- 双击 ai_tile_mvp/启动AI采样.bat

采样建议：

- 先采 300 到 500 张
- 不要连拍完全相同的镜头
- 同一场景每隔 8 到 15 帧保留 1 张更合适
- 覆盖缩放、动画、边缘透视、复杂背景四类困难场景

## 5. 安装 AnyLabeling

### 方案 A：装到你当前启动工作台的 Python 环境里

这是最省事的方式。装完后，工作台里的“启动 AnyLabeling”通常可以自动找到它。

在和工作台同一个 Python 环境里执行：

```powershell
python -m pip install -U uv
python -m uv pip install x-anylabeling-cvhub[cpu]
```

安装后验证：

```powershell
xanylabeling checks
```

### 方案 B：下载绿色版 EXE

如果你不想装到当前 Python 环境，也可以去官方发布页下载 Windows 压缩包。

建议做法：

1. 下载 X-AnyLabeling 的 Windows 发布包
2. 解压到你自己习惯的位置
3. 回到工作台“标注与抽检”页
4. 在“AnyLabeling 启动程序”里手动选中 xanylabeling.exe
5. 之后点“启动 AnyLabeling”即可

## 6. 最省事的启动方式

推荐你直接这样做：

1. 打开 ai_tile_mvp/workbench.py 或双击 启动AI工作台.bat
2. 进入“标注与抽检”页
3. 点击“启动 AnyLabeling”
4. 再点“打开属性配置”
5. 在 AnyLabeling 里执行 Upload -> Upload Attributes File
6. 选中 configs/resource_node_attributes.json

工作台会自动帮你带入：

- 图片目录：datasets/plot_det/raw/images
- 标签目录：datasets/plot_det/raw/labels
- 检测类别文件：configs/label_classes.txt

## 7. AnyLabeling 里具体怎么点

### 第一次打开后先确认 3 件事

1. 左侧已经加载的是 raw/images 目录
2. 标注输出目录是 raw/labels
3. 可选类别里只有 resource_node

### 开始标单张图

1. 在左侧文件列表里点击一张图片
2. 选择矩形框工具，也就是 Rectangle / 矩形框
3. 在一个菱形资源点外沿拖出一个框
4. 类别统一选择 resource_node
5. 在右侧属性面板里补 3 个属性：等级、类型、关系
6. 继续标下一块

如果你当前这版 X-AnyLabeling 在导入属性文件后会崩，就先不要导入属性文件，直接在画框弹窗里的 description 文本框中写“5级 木材 敌对”这种内容也可以。当前同步脚本已经支持从 description 自动解析等级、类型、关系。

提高效率的关键点：

- 先把一整张图的框画完
- 再用 Loop Select Shapes 逐个切换 shape 补属性
- 这样比“画一个框就选一次 64 类”快很多

### 标注时只记一个原则

你标的是“菱形目标框”，不是只框中间数字。

也就是：

- 框里要包含完整菱形主体
- 框不要大到把相邻菱形一起吃进去
- 框也不要小到只剩数字或图标

### 一张图标完后怎么保存

如果界面已经自动保存，就继续下一张。

如果你不确定有没有保存：

1. 手动点一次保存
2. 去 raw/labels 看看有没有同名 .json 文件

这里的 .json 是核心标注文件，不要删。

## 8. 哪些菱形要标，哪些不要标

要标：

- 主体清晰可判断的菱形
- 有轻度动画但主体仍可判断的菱形
- 有轻度遮挡但主体仍可判断的菱形
- 画面边缘有透视但主体仍可判断的菱形

不要标：

- 主体已经出画的菱形
- UI 或特效大面积遮住，导致主体看不清的菱形
- 只是纹理碎片、边角阴影或不完整残片的区域

## 9. 标完以后怎么生成训练数据

这一步不再要求你手动导出 YOLO。

现在直接运行同步脚本，它会自动把 AnyLabeling JSON 变成两类数据：

1. 检测用 txt 标签
2. 三个属性分类任务的裁剪图

命令如下：

```powershell
python ai_tile_mvp/scripts/sync_resource_annotations.py --image-dir ai_tile_mvp/datasets/plot_det/raw/images --json-dir ai_tile_mvp/datasets/plot_det/raw/labels --detection-label-dir ai_tile_mvp/datasets/plot_det/raw/labels --attr-root ai_tile_mvp/datasets/plot_attr_cls --clear-attr-raw
```

生成后你会得到：

- raw/labels/*.txt: 单类检测标签
- datasets/plot_attr_cls/level/raw/*: 等级分类原始裁剪集
- datasets/plot_attr_cls/resource_type/raw/*: 类型分类原始裁剪集
- datasets/plot_attr_cls/relation/raw/*: 关系分类原始裁剪集

## 10. 导出后快速自检

检测抽检仍然可以这样做：

```powershell
python ai_tile_mvp/scripts/check_yolo_labels.py --image-dir ai_tile_mvp/datasets/plot_det/raw/images --label-dir ai_tile_mvp/datasets/plot_det/raw/labels --output-dir ai_tile_mvp/outputs/label_check --sample-count 40
```

重点看四件事：

- 框是否稳定包住完整菱形
- 是否有漏标一整片区域
- 框大小是否忽大忽小
- JSON 里的属性有没有漏填

## 11. 什么时候可以开始训练

满足下面几条就可以开始首轮训练：

- raw/images 和 raw/labels 一一对应
- raw/labels 里已经有对应的 .json 和 .txt
- 抽检 40 张没有大面积漏框或乱框
- 属性裁剪集里三个任务都已经生成出样本

## 12. 训练前最后一步

检测切分：

```powershell
python ai_tile_mvp/scripts/split_yolo_dataset.py --source-images ai_tile_mvp/datasets/plot_det/raw/images --source-labels ai_tile_mvp/datasets/plot_det/raw/labels --output-root ai_tile_mvp/datasets/plot_det --clear-output
```

属性切分，以等级任务为例：

```powershell
python ai_tile_mvp/scripts/split_attribute_classification_dataset.py --source-raw-root ai_tile_mvp/datasets/plot_attr_cls/level/raw --output-root ai_tile_mvp/datasets/plot_attr_cls/level --clear-output
```

然后再分别启动检测训练和属性训练脚本。
