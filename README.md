# WindowPilot - 窗口自动化与识别工具

一个基于 Python 和 PySide6 的 Windows 桌面自动化工具，围绕目标窗口提供截图、识别、输入、录制回放、坐标映射、任务编排，以及网络与内存辅助分析能力。

## 适用场景

- 需要对指定窗口执行重复点击、输入、拖动和流程编排
- 需要通过模板匹配或 OCR 定位界面元素
- 需要将屏幕坐标映射为逻辑坐标，并在六方格场景中做遍历或路径规划
- 需要抓取目标进程网络消息或扫描内存数值
- 需要将传统模板识别逐步升级到可选的 AI 目标识别链路

## 功能概览

### 1. 目标窗口与截图

- 枚举可见窗口，选择目标窗口并查看预览
- 支持客户区截图、后台截图、前台截图等多种路径
- 适合先做单点验证，再逐步接入录制、识别或任务流程
- 窗口大小、热键和坐标轴配置会自动持久化

### 2. 脚本录制与回放

- 录制鼠标点击、移动和键盘事件
- JSON 格式存储，可编辑、可重复回放
- 支持倍速、循环、暂停、紧急停止
- 适合快速验证简单流程

### 3. 图像识别

- 支持导入模板图像，或直接截图生成模板
- 提供普通模板匹配与前景优先匹配两种模式
- 支持多尺度匹配、透明通道掩码、颜色一致性复核
- 可在匹配后直接点击，支持后台模式
- 适合静态图标、按钮、资源点、动画帧等目标

### 4. 文字识别

- 基于 RapidOCR 的窗口文字识别
- 支持最小置信度过滤、关键字筛选
- 识别结果可双击点击，也可对选中文字位置执行滑动
- 支持保存当前截图用于排查识别问题

### 5. 坐标转换

- 对当前窗口截图进行手工标定
- 支持添加坐标点、辅助线、逻辑锚点和轴线校准
- 支持坐标点 CSV 导入导出
- 保存完整工作状态到 `coordinate_workspace_state.json`，同时保存映射到 `coordinate_mappings.json`
- 关联截图会保存到 `coordinate_workspace_assets/`
- 输出的相对坐标与逻辑坐标可被计划任务直接复用

### 6. 计划任务

- 可视化编辑任务、步骤、条件和失败操作
- 支持变量、坐标、图像、数组、坐标数组、结构体、结构体数组
- 支持循环步骤、跳转步骤、单步测试、导入导出、日志复制
- 识别类型支持：文字识别、单图像模板匹配、多图像识别、可选 AI 目标识别
- 操作类型覆盖：点击、双击、右键、长按、输入文本、按键、拖动、红框标记、数组写入、识别坐标转逻辑坐标、网格遍历、周围坐标展开、铺路路径查找等
- 支持自定义识别范围、匹配序号、多匹配、高亮时长、失败后补救动作等高级配置

### 7. 网络消息

- 按目标进程 PID 抓取网络消息
- 支持协议、方向、端口、关键字过滤
- 内置 Hex 视图、详情视图、数值搜索、偏移解析
- 支持保存消息到 JSON、重新加载、编辑后发送、新建发送
- 安装 Npcap 后可获得更完整的双向抓取能力；未安装时通常只能抓到发送方向

### 8. 内存扫描

- 附加目标进程后按数值做首次扫描和再次扫描
- 支持等于、不等于、大于、小于、值变化、值未变化等筛选方式
- 可将地址加入实时监控列表并持续刷新数值
- 适合定位坐标、计数器、状态标志等内存值

### 9. 系统集成

- 全局热键默认值：`F9` 开始或停止，`F10` 暂停或继续，`F12` 紧急停止
- 支持托盘最小化
- 设置页可修改热键和窗口启动行为

## 环境要求

- Windows 10/11
- Python 3.9+
- 建议使用 64 位 Python 和虚拟环境
- 某些功能需要管理员权限，例如后台输入、完整网络抓取、内存附加与扫描

## 安装

### 1. 安装基础依赖

```bash
cd game_assistant
pip install -r requirements.txt
```

基础依赖包括：

- `PySide6`
- `mss`
- `opencv-python`
- `numpy`
- `Pillow`
- `pywin32`
- `pynput`
- `pystray`
- `rapidocr-onnxruntime`
- `scapy`
- `psutil`

### 2. 可选增强组件

- 若需要更完整的网络抓取，请安装 Npcap，并在安装后重启程序
- 若需要 AI 目标识别工作台，请额外安装 `ai_tile_mvp/requirements-ai.txt`
- 若需要标注工具，可参考工作区中的 `X-AnyLabeling/` 目录

## 推荐使用流程

### 方案一：快速录制回放

1. 选择目标窗口
2. 打开“脚本录制”页签
3. 开始录制并执行一遍操作
4. 保存脚本
5. 调整速度和循环次数后回放

### 方案二：模板或 OCR 驱动的自动化

1. 选择目标窗口
2. 在“图像识别”中添加模板，或在“文字识别”中读取文字
3. 先用“测试识别”或文字点击验证定位是否准确
4. 如果只需要单次动作，可直接在识别页执行
5. 如果需要完整流程，再转到“计划任务”把识别与操作串起来

### 方案三：坐标驱动的任务编排

1. 在“坐标转换”中截取当前窗口并完成点位标定
2. 保存当前状态并导出坐标点 CSV
3. 在“计划任务”中使用识别坐标转逻辑坐标、网格遍历、路径查找等动作
4. 执行任务并根据日志迭代参数

### 方案四：网络或内存排查

1. 选择目标窗口
2. 切到“网络消息”或“内存扫描”页签
3. 先确认 PID 已正确关联
4. 再开始抓取、搜索、筛选或监控

## 关键数据文件

- `config.json`：窗口、热键、识别和坐标轴基础配置

## 常见问题

### 1. 后台点击或后台截图没有效果

- 某些目标程序对子窗口、权限级别或绘制方式有额外限制
- 可尝试提升程序权限，或改用前台模式验证流程
- 对拖动和点击类动作，先在“图像识别”页单独测试更容易定位问题

### 2. 网络面板只能看到发送消息

- 这通常是未安装 Npcap
- 安装后重启程序，再重新开始抓取

### 3. 内存扫描无法附加

- 先确认目标进程 PID 是否正确
- 再确认当前程序是否以管理员身份运行
- 某些受保护进程可能仍然拒绝附加

### 4. 坐标转换保存后效果不稳定

- 优先使用客户区相对坐标
- 重新检查逻辑锚点和轴线方向
- 保证导出的 CSV 和当前窗口标定来自同一套视角
